# seedsplit on Windows (PowerShell, BETA)

A PowerShell port of [`seedsplit`](../README.md) — split a secret (seed phrase /
password / key) into Shamir shares, any `T` of which reconstruct it, while `T-1`
reveal nothing.

> **BETA.** The logic is covered by Pester, **including a known-answer test that
> reconstructs a share-set generated by the macOS (bash) build** — so shares are
> byte-compatible across platforms (split on a Mac, combine on Windows, or vice
> versa). What is *not* yet broadly field-tested: exotic console code pages and
> locales. Verify on a throwaway secret before trusting a real seed phrase.

## Cross-platform shares

seedsplit's wire format is identical on both builds:

```
SSS2-<set-id>-<T>-<x>-<hexY>-<chk>
```

Same GF(256) field (generator `0x03`, reducing polynomial `0x11b`), same integrity
wrapper (`0x55 | len | secret | 16-byte SHA-256 tag`), same per-share checksum. A
share file moves between macOS and Windows unchanged.

## Install (verify-then-run)

Requires [PowerShell 7+](https://aka.ms/powershell) (`pwsh`); Windows PowerShell 5.1
also runs the script.

```powershell
irm https://github.com/Di-kairos/seedsplit/releases/latest/download/install.ps1 -OutFile install.ps1
irm https://github.com/Di-kairos/seedsplit/releases/latest/download/SHA256SUMS  -OutFile SHA256SUMS
# verify install.ps1's hash against SHA256SUMS, read the script, then:
pwsh -File install.ps1
```

The installer downloads `seedsplit.ps1` + `SHA256SUMS` from the **release tag**,
verifies the SHA-256 **before** installing (fail-closed on mismatch or missing
entry), drops the script into `%LOCALAPPDATA%\Programs\seedsplit`, writes a
`seedsplit.cmd` shim, and adds that folder to your user `PATH`.

Open a new terminal afterward so `PATH` refreshes.

## Commands

| Command | What it does |
|---------|--------------|
| `seedsplit split [-n N] [-t T] [--file F]` | Split a secret (from stdin or `--file`) into `N` shares; any `T` reconstruct it. Default `-n 3 -t 2`. |
| `seedsplit combine [FILE...]` | Reconstruct the secret from `≥T` shares (stdin, one per line, or from files). |
| `seedsplit verify  [FILE...]` | Confirm `≥T` shares reconstruct the secret **without printing it**. |
| `seedsplit version` | Show the version. |

```powershell
"correct horse battery staple" | seedsplit split -n 5 -t 3 > shares.txt
Get-Content shares.txt | Select-Object -Index 0,2,4 | seedsplit combine
```

The secret is read from **stdin or `--file`, never from arguments** (arguments are
visible in the process list). `ST_LANG=ru` switches messages to Russian.

## Scope & limitations (honest)

- Share quality = RNG quality. The port uses the OS cryptographic RNG
  (`RandomNumberGenerator`), not a homegrown source.
- A threshold protects against leaking `< T` shares; it does **not** protect against
  *losing* `≥ (N−T+1)` shares. Store and distribute shares across independent places.
- **No SLIP-39 / hardware-wallet interop yet** — a deliberate scope decision for a
  later pack.
- GF(256) multiply uses log/antilog tables — **not constant-time** (a timing
  side-channel), out of scope for a local CLI but not hidden.
