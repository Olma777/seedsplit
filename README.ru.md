# seedsplit

Распределить секрет на доли (Shamir Secret Sharing) — без снейкойла про «надёжность».

[![CI](https://github.com/Di-kairos/seedsplit/actions/workflows/ci.yml/badge.svg)](https://github.com/Di-kairos/seedsplit/actions/workflows/ci.yml)
![License: MIT](https://img.shields.io/badge/license-MIT-green)
![platform](https://img.shields.io/badge/platform-macOS-blue)

Часть экосистемы [Paranoid Tools](https://github.com/Di-kairos/paranoid-tools).

## Зачем это

Разбить seed-фразу, пароль или ключ на N долей так, что любые T из них восстанавливают
секрет, а T-1 долей не дают о нём **ничего**. Чтобы один носитель или бэкап не был
единой точкой отказа и компрометации: потеряли один лист — секрет цел; нашли один
лист — узнать ничего нельзя.

## Установка

Установщик тянет бинарь **и `SHA256SUMS` с релизного тега** (а не с подвижной ветки
`main`) и проверяет контрольную сумму **до** установки — fail-closed при любом
несовпадении.

### Verify-then-run (не доверяй — проверяй)

`curl | bash` означает запуск кода, который ты не читал. Лучше так — скачать, сверить
сумму, прочитать глазами, потом запустить:

```bash
base=https://github.com/Di-kairos/seedsplit/releases/latest/download
curl -fsSLO "$base/install.sh"
curl -fsSLO "$base/SHA256SUMS"
shasum -a 256 -c SHA256SUMS --ignore-missing   # проверяет install.sh
less install.sh                                  # прочитать
bash install.sh
```

### Установка одной строкой

```bash
curl -fsSL https://github.com/Di-kairos/seedsplit/releases/latest/download/install.sh | bash
```

> **Целостность ≠ подлинность (честные границы).** Контрольная сумма доказывает, что
> скачанный бинарь совпадает с `SHA256SUMS` из **того же релиза** — это ловит
> повреждение, частичную/кэш-подмену и не даёт запустить код с подвижной ветки `main`.
> Она сама по себе НЕ защищает от атакующего, способного переписать *и* бинарь, *и* его
> сумму в источнике (или на твоём канале), и НЕ доказывает, *кто* их опубликовал. Для
> воспроизводимости фиксируй версию: `SEEDSPLIT_VERSION=0.3.1` вместо `latest`. Источник
> можно переопределить через `SEEDSPLIT_BASE_URL`, путь установки — через `SEEDSPLIT_DEST`.

### Язык вывода

По умолчанию английский. Для русского — `ST_LANG=ru` (русская системная локаль
подхватывается автоматически).

## Использование

Секрет читается из **stdin** или `--file`, **никогда** из argv (argv виден в `ps`).

Разбить секрет на 5 долей с порогом 3 (любые 3 восстановят, любые 2 не дадут ничего):

```bash
$ printf '%s' "legal winner thank year wave sausage worth useful legal winner thank yellow" \
    | seedsplit split -n 5 -t 3
SSS2-b552023e-3-1-5b5a5efd9de59461…-48d2
SSS2-b552023e-3-2-ed9f531596633fc8…-2f21
SSS2-b552023e-3-3-e3c529846ee1cac5…-f18f
SSS2-b552023e-3-4-9889f085f80357f1…-311f
SSS2-b552023e-3-5-96d38a140081a2fc…-b91b
```

Каждая строка — самостоятельная доля формата `SSS2-<setid>-<T>-<x>-<hexY>-<chk4>`:
версия формата, случайный **set-id**, общий для всех долей одного сплита (поэтому
`combine` детерминированно отказывается смешивать доли из *разных* сплитов), порог,
номер точки, тело и 4-символьная контрольная сумма (ловит опечатку в конкретной доле).
Разнеси строки по разным носителям/местам.

Восстановить секрет — подай **любые T (или больше) долей** на stdin, по одной на строку:

```bash
$ head -3 shares.txt | seedsplit combine
legal winner thank year wave sausage worth useful legal winner thank yellow
```

Можно подать доли и файлами-аргументами:

```bash
seedsplit combine share1.txt share2.txt share3.txt
```

Проверить, что набор восстанавливается, **не раскрывая секрет** (удобно при раскладке
долей по носителям — подтвердить корректность, ничего не показывая). Выходит с кодом `0`
и печатает только длину восстановленного секрета, но не сам секрет:

```bash
$ head -3 shares.txt | seedsplit verify
✓ verify: доли согласованы, секрет восстановим (36 байт). Секрет НЕ показан.
```

Версия и справка:

```bash
seedsplit version        # или -v / --version
seedsplit help           # или -h / --help
```

### Команды и флаги

| Команда | Что делает |
|---|---|
| `seedsplit split [-n N] [-t T] [--file F]` | Разбить секрет (из stdin или `--file`) на `N` долей; любые `T` восстанавливают его. По умолчанию `-n 3 -t 2`. |
| `seedsplit combine [FILE...]` | Восстановить секрет из ≥T долей (читает из stdin по строке на долю, либо из `FILE`). |
| `seedsplit verify [FILE...]` | Подтвердить, что ≥T долей восстанавливаются, **не печатая секрет** (печатает только длину восстановленного секрета). |
| `seedsplit version` | Показать версию (также `-v` / `--version`). |
| `seedsplit help` | Показать справку (также `-h` / `--help`). |

Ограничения параметров: порог `-t` должен быть ≥2 (иначе одна доля = весь секрет),
`-n` должно быть ≥ `-t` и ≤255 (точки оценки в GF(256)); секрет — до 65535 байт.

## Как это работает

Чистая реализация **Shamir Secret Sharing над GF(256)** (поле `2^8`, редуцирующий
многочлен `0x11b` как в AES; сложение = XOR, умножение через таблицы log/antilog от
генератора `0x03`). Случайные коэффициенты многочленов берутся из `/dev/urandom`.

Секрет оборачивается заголовком целостности (`0x55` | длина | секрет | **128-битный
tag** из sha256), поэтому `combine` возвращает **либо точный секрет, либо честный
отказ** — шанс молча вернуть *не тот* секрет ≈ 2⁻¹²⁸. Каждая доля также несёт случайный
**set-id** (4-байтный nonce, *не* выведенный из секрета — так что его нельзя использовать
как оракул подтверждения догадки), благодаря чему `combine` детерминированно отвергает
смесь долей из разных сплитов. Отказы различимы и конкретны: порченная доля (контрольная
сумма), доли из разных сплитов (set-id), ниже порога, несогласованный порог или
провалившаяся проверка целостности.

## Область применения и ограничения

Честность — смысл всей экосистемы; здесь особенно легко скатиться в снейкойл, поэтому
вот честные пределы:

- **качество долей = качество ГСЧ** — берём `/dev/urandom`, не самопал;
- **секрет в `argv` виден в `ps`** — ввод только через stdin/файл, не аргументом;
- **доли безопасны ровно настолько, насколько безопасно ты их ХРАНИШЬ и РАЗНОСИШЬ;**
  порог защищает от утечки <T долей, но не от ПОТЕРИ ≥(N-T+1) долей — это твоя
  ответственность за географию и носители;
- **совместимости со SLIP-39 / аппаратными кошельками ПОКА НЕТ** — полноценный SLIP-39
  (1024-словарь + RS1024 + шифрование) это отдельное решение по объёму, не «ноль
  зависимостей»; честно помечено как scope-решение на будущий пак;
- **умножение в GF(256) идёт через таблицы log/antilog и НЕ constant-time** — есть
  тайминговый side-channel. Он вне модели угроз этого инструмента (локальный CLI без
  удалённого или онлайн-оракула на `split`/`combine`), но мы его не скрываем.

## Архитектура

- Single-file Bash, ноль зависимостей. Shamir над GF(256) реализован на чистом Bash;
  ГСЧ — `/dev/urandom`. Тесты: **bats** (37 тестов — `split`/`combine`/`verify`,
  round-trip по каждому пороговому подмножеству, полная таксономия отказов, 128-битный
  tag целостности, плюс known-answer тесты: GF-векторы FIPS-197 и замороженный набор
  долей).
- Общее ядро (`lib/common.sh`) **вендорится** из securetrash inline, пиннуто к git-ref;
  `tools/vendor-common.sh --check` ловит дрейф в CI. См. `paranoid-tools/README.md`.

## Windows (beta)

PowerShell-порт уже существует — в [`windows/README.md`](windows/README.md). Он повторяет
логику macOS — тот же Shamir над GF(256), `RNGCryptoServiceProvider` вместо `/dev/urandom` —
и даёт **байт-совместимые** доли (раздели на одной ОС, собери на другой).

> **Beta:** Windows-порт протестирован по логике (Pester на CI), но ещё не проверен на
> реальном Windows-железе. См. [`windows/README.md`](windows/README.md).
